## 1. Agile & Scrum for Learning Product Development

### Why Agile for L&D?
Traditional ADDIE is waterfall — sequential phases with handoffs. This works when requirements are stable, but modern L&D rarely has that luxury. Agile addresses:
- **Changing requirements**: Business needs shift mid-project; Agile accommodates this
- **Faster time-to-value**: Deliver usable learning increments every sprint, not after 6 months
- **Continuous improvement**: Build feedback loops into the process, not just at the end
- **Stakeholder alignment**: Regular sprint reviews keep sponsors engaged and prevent "wrong direction" surprises

### Core Scrum Roles — L&D Adaptation

| Scrum Role | L&D Equivalent | Responsibilities |
|---|---|---|
| **Product Owner** | L&D Project Lead / ID | Owns backlog priority, defines acceptance criteria, represents learner and business |
| **Scrum Master** | L&D Scrum Master / PM | Removes impediments, facilitates ceremonies, protects team from scope creep |
| **Development Team** | ID team (IDs, graphic designers, SMEs, developers) | Commits to sprint work, self-organizes, delivers working learning increments |

**Coaching note**: SMEs are rarely on the Scrum team full-time — they are stakeholders. Manage SME availability as a dependency, not an assumption. Factor SME review cycles into sprint planning explicitly.

### Sprint Structure for L&D

**Recommended sprint length**: 2 weeks for most L&D projects (1 week for rapid iterations; 3 weeks for complex blended programs)

**Definition of Done for a Learning Increment**:
- Storyboard reviewed and approved by SME
- Course/module built and QA'd in authoring tool
- Published to staging LMS and tested (SCORM tracking verified)
- Accessibility check completed
- Peer-reviewed by another ID
- Accepted by Product Owner

## 2. Adapting Scrum Ceremonies for L&D

### Sprint Planning
- Review top-priority backlog items with the team
- Break epics into stories; stories into tasks with time estimates
- Identify dependencies (SME availability, asset delivery, LMS access)
- Set sprint goal: "By end of this sprint, learners will be able to access Module 2 on the staging LMS"
- Capacity planning: account for SME review lag — don't fill 100% capacity

### Daily Standup (15 min max)
Three questions:
1. What did I complete yesterday toward the sprint goal?
2. What will I do today?
3. What is blocking me?

**L&D-specific blockers to surface early**: Waiting on SME feedback, asset approval delays, LMS access issues, unclear scope from sponsor.

### Sprint Review (Demo)
- Show working learning increments — actual course modules, not slides about modules
- Include stakeholders and sponsors: "Here is what learners will experience"
- Collect structured feedback: What works? What needs adjustment? What's next priority?
- Update backlog based on feedback before next sprint

### Sprint Retrospective (Team-only)
Three columns:
- **What went well** — protect and amplify
- **What didn't go well** — diagnose root cause
- **What we'll try next sprint** — one or two concrete experiments

**Common L&D retro themes**: SME bottlenecks, scope creep from stakeholders, tool/tech issues, unclear acceptance criteria, insufficient review time budgeted.

### Backlog Refinement (mid-sprint, ~1 hour)
- Groom upcoming stories: clarify, estimate, split large items
- Ensure next sprint's top items are "ready" — clear, estimated, and accepted criteria defined
- Add new items as they emerge; re-prioritize with Product Owner

## 3. Writing Learning Product Backlogs

### Hierarchy
**Epic** → **User Story** → **Task**

### Writing L&D User Stories
Format: *As a [learner/stakeholder], I want [capability], so that [outcome].*

Examples:
- "As a new sales rep, I want to practice a difficult customer objection scenario, so that I feel confident handling it in the field."
- "As a compliance officer, I want a dashboard showing which employees have completed mandatory training, so that I can report to the audit team."
- "As a learner, I want the course to remember where I left off, so that I don't have to restart when I return."

### Acceptance Criteria
Each story needs explicit, testable criteria. Example:
- Story: "As a learner, I want scenario-based practice with feedback."
- Acceptance criteria:
  - Three branching scenarios present, each with ≥2 decision points
  - Incorrect choices redirect to consequence + coaching, not just "wrong, try again"
  - Correct path provides positive reinforcement with explanation
  - SME has reviewed and approved all scenario content
  - Scenario passes QA checklist (tab order, audio sync, SCORM tracking)

### Backlog Item Types for L&D

| Type | Examples |
|---|---|
| **Content development** | Write storyboard for Module 3, build branching scenario in Storyline |
| **Asset creation** | Design custom icons, record/edit video, create infographic |
| **Technical** | Configure LMS course shell, set up xAPI tracking, accessibility remediation |
| **Review/approval** | SME review cycle, legal review, accessibility audit |
| **Infrastructure** | Establish design system, build master slide template, create style guide |

## 4. SAM + Scrum: Where They Meet

Allen Interactions' SAM (Successive Approximation Model) is already iterative — it maps well onto Scrum:

| SAM Phase | Scrum Parallel |
|---|---|
| Preparation Phase | Sprint 0 — information gathering, audience analysis, project setup |
| Iterative Design (savvy start) | Sprint 1–2 — rapid prototype, stakeholder review, early course shell |
| Iterative Development | Sprints 3–N — build → review → refine cycles |
| Alpha / Beta / Gold | Sprint Reviews + final acceptance sprint |

**Key alignment**: Both embrace "fail early, fail cheap." The savvy start in SAM is essentially a design sprint — rapid sketching, prototyping, getting something tangible in front of stakeholders fast.

**Where they diverge**: SAM's phases are more fluid and overlap freely; Scrum has strict sprint boundaries. In practice, most L&D teams run a "SAM-flavored Scrum" — Scrum structure with SAM's iterative prototype philosophy.

## 5. Agile L&D — Common Pitfalls

- **Treating SME review as a task, not a dependency**: SME review is external — it can blow up a sprint. Buffer for it explicitly.
- **Skipping retrospectives**: The retrospective is where teams get better. Never skip it.
- **Waterfall backlog**: Writing all stories for the whole project upfront defeats Agile's purpose. Keep a rolling, prioritized backlog — detail only what you'll build in the next 2 sprints.
- **No Definition of Done**: Without agreed acceptance criteria, "done" means different things to everyone. Define it together, in writing, before Sprint 1.
- **Agile theater**: Running standup and calling it Agile. True Agile requires empirical inspection, adaptation, and a working increment every sprint — not just the ceremonies.
- **Ignoring technical debt**: In eLearning, this is template inconsistency, accessibility gaps, un-QA'd interactions. Address it in each sprint; don't defer it all to the end.

## 6. Visual Design Principles for Learning

### Four Core Visual Design Principles (from Robin Williams' *The Non-Designer's Design Book*)

These four principles — widely known by the acronym CRAP — are foundational to visual communication and are factual design concepts in the public domain. Williams named and popularized them; they are not proprietary, but the original book is the canonical reference.

- **Contrast**: Make different things look different. Same-size, same-color elements read as equal weight. Use contrast to signal hierarchy.
- **Repetition**: Repeat visual elements (colors, fonts, shapes, spacing) to create unity and reinforce brand.
- **Alignment**: Every element should have a visual connection to something else. Avoid arbitrary placement.
- **Proximity**: Group related items. Separate unrelated items. Proximity creates relationship.

### Gestalt Principles Applied to eLearning
- **Figure/Ground**: Ensure interactive elements stand out from background — low contrast clickables cause usability failures
- **Similarity**: Use consistent visual treatment for elements of the same type (all buttons same style, all headers same font/size)
- **Continuity**: Guide the eye through content with visual flow — left to right, top to bottom, or explicitly guided with arrows/lines
- **Closure**: Learners complete incomplete shapes — use this for progress indicators, interactive reveals
- **Proximity**: Group instructional content + related practice activity visually; separate feedback from question stem

### Visual Hierarchy for Learning
Layer 1 (first attention): Learning objective / key question — largest, highest contrast
Layer 2 (second attention): Primary instructional content — clear, readable, supported by visuals
Layer 3 (supporting): Examples, captions, labels — smaller, lower contrast
Layer 4 (peripheral): Navigation, branding — should not compete with content

### Whitespace is Not Wasted Space
Cognitive load research is clear: dense slides impair learning. Every element added to a screen competes for working memory. Whitespace = breathing room = reduced extraneous load. Coach users away from the "fill the slide" instinct.

## 7. UX Design for eLearning

### Nielsen's 10 Usability Heuristics — Applied to eLearning

1. **Visibility of system status**: Show progress — progress bar, lesson counter, completion indicators
2. **Match between system and real world**: Use learner's language, not L&D jargon; familiar metaphors
3. **User control and freedom**: Allow back navigation; don't trap learners in linear paths without escape
4. **Consistency and standards**: Same button does the same thing everywhere; consistent terminology
5. **Error prevention**: Don't let learners submit an assessment without answering all questions; confirm destructive actions
6. **Recognition over recall**: Keep navigation visible; don't make learners remember where things are
7. **Flexibility and efficiency**: Allow skipping for experienced learners; provide keyboard shortcuts
8. **Aesthetic and minimalist design**: Every element on screen should earn its place
9. **Help users recognize, diagnose, and recover from errors**: Clear error messages with corrective guidance
10. **Help and documentation**: Provide orientation for complex simulations; tooltips for ambiguous interactions

### UX Research Methods for Learning Products
- **Usability testing**: 5 users reveal ~85% of usability problems (Nielsen). Test early prototypes, not finished courses.
- **Think-aloud protocol**: Ask learners to narrate what they're thinking as they navigate — surfaces confusion invisible to the designer
- **Card sorting**: For navigation and content organization — let learners group topics to validate your information architecture
- **First-click testing**: Where do learners click first? Validates interface affordances
- **Heuristic evaluation**: Expert review against Nielsen's heuristics — fast and cheap, catches obvious issues

### Interaction Design Patterns for eLearning
- **Affordances**: Buttons should look like buttons; clickable images should signal interactivity (subtle hover state, cursor change)
- **Feedback loops**: Every interaction needs immediate feedback — state changes, sounds, confirmations
- **Progressive disclosure**: Don't reveal everything at once; expand/reveal patterns reduce overwhelm
- **Mental models**: Design to match how learners expect something to work based on prior experience

## 8. UI Design for Learning Interfaces

### eLearning Interface Zones
- **Header/Branding**: 8-12% of screen — consistent, non-distracting
- **Content Area**: 65-75% of screen — primary learning real estate
- **Navigation**: 10-15% — persistent, predictable, accessible
- **Feedback/Status**: Contextual — appears when needed, dismissible

### Button Design for eLearning
- Minimum touch target: 44×44px (WCAG mobile standard)
- Clear affordance: shape, color, and label must collectively communicate "click me"
- State design: Normal → Hover → Active/Pressed → Disabled — design all four
- Hierarchy: Primary action (Continue) > Secondary action (Review) > Tertiary (Exit/Menu)
- Label with verbs: "Continue" not "Next"; "Check Answer" not "Submit"; "Try Again" not "Retry"

### Navigation Patterns
- **Linear**: Best for compliance, sequential skill-building where order matters
- **Menu-based**: Best for reference content, job aids, learner-controlled exploration
- **Branching**: Best for scenario-based learning, soft skills, decision-making practice
- **Hub-and-spoke**: Main menu returns between activities — good for modular programs
- Never remove the ability to exit. Always provide orientation ("You are here").

## 9. Typography in Learning Design

### Font Selection Principles
- **Serif fonts** (Georgia, Garamond): Higher readability in long-form body text; conveys authority and tradition
- **Sans-serif fonts** (Inter, Open Sans, Lato): Superior screen readability; preferred for eLearning UI
- **Display fonts**: Headlines only; never body text; high personality, low legibility at small sizes
- **Monospace** (Courier, Consolas): Code examples only

### Type Scale for eLearning
- H1 (Slide title): 28-36px, bold
- H2 (Section header): 22-26px, semi-bold
- Body: 16-18px, regular (never below 14px for accessibility)
- Caption/label: 12-14px, regular or medium
- Button: 14-16px, medium or bold

### Readability Rules
- **Line length**: 50-75 characters per line (optimal); never exceed 90
- **Line height**: 1.4-1.6× font size for body text
- **Letter spacing**: 0 for body; slightly positive for all-caps labels
- **Contrast**: Body text must meet WCAG AA (4.5:1 against background); large text 3:1
- **Avoid**: All-caps for body text, light fonts on light backgrounds, more than 2 font families

## 10. Color Theory Applied to Learning

### Color in Learning — Functional Roles
- **Brand color**: Establishes identity and consistency; typically in header, primary buttons, accents
- **Semantic color**: Consistent meaning across interactions — green = correct/success, red = error/incorrect, yellow = caution/warning, blue = information
- **Hierarchy color**: Higher contrast = more important; use color weight to guide attention
- **Emotional color**: Warm tones (orange, red) = energy, urgency; cool tones (blue, green) = calm, trust; purple = creativity, sophistication

### Accessibility Color Rules
- Never use color as the ONLY indicator — always pair with icon, label, or pattern
- Color contrast: 4.5:1 for normal text; 3:1 for large text and UI components (WCAG AA)
- Test with color blindness simulators (Coblis, Adobe Color accessibility tools, Stark plugin)
- ~8% of males have some form of color vision deficiency — red/green dependency is a common failure

### Building a Learning Color Palette
- **Primary**: Brand color — used for key UI elements, primary actions
- **Secondary**: Complementary or analogous — supporting elements
- **Neutral scale**: 5-7 grays from near-white to near-black — backgrounds, text, borders
- **Semantic**: 4 colors (success, error, warning, info) — consistent throughout
- **Dark/Light modes**: Design for both if your LMS or platform supports it

## 12. Adobe Photoshop for L&D

### Core Skills for L&D Use
- **Non-destructive editing**: Always use adjustment layers, smart objects, and smart filters — never edit pixels directly
- **Masking**: Layer masks and clipping masks for clean, re-editable cutouts; avoid erasing
- **Smart Objects**: Embed Illustrator vectors or other assets as smart objects — scale without quality loss
- **Batch processing**: Actions + Image Processor for bulk resizing/exporting course images

### Key L&D Workflows

**Custom screenshot preparation:**
1. Take screenshot at 1:1 (no scaling)
2. Place in Photoshop as smart object
3. Add callout shapes on separate layers (rounded rectangles, arrows)
4. Use Layer Styles for callout shadows and strokes
5. Export → Export As PNG (not Save for Web — deprecated)

**Person/object cutout for eLearning characters:**
1. Select with Object Selection tool (AI-powered in CC)
2. Refine edge: Select and Mask → Smart Radius, Decontaminate Colors
3. Output to: New Layer with Layer Mask
4. Place on white or transparent background for authoring tool import

**Exporting for eLearning:**
- PNG for graphics with transparency (UI elements, characters, illustrations)
- JPG for photographic backgrounds (high quality, 85-90%)
- SVG export: Not native in Photoshop — use Illustrator instead
- Resolution: 144ppi for retina screens; 72ppi minimum; export at 2× and let HTML/CSS scale down

## 13. Adobe Illustrator for L&D

### Why Illustrator Over Photoshop for L&D Assets
- Vector = infinite scalability, small file sizes, sharp on any screen
- Better for: icons, diagrams, infographics, process flows, custom characters, logos
- SVG export allows direct use in HTML/CSS and some authoring tools

### Core L&D Workflows

**Custom icon creation:**
1. Start with basic geometric shapes (no drawing required for most icons)
2. Use Pathfinder (Unite, Minus Front, Intersect) to combine/subtract shapes
3. Stroke consistency: use consistent stroke weights within an icon set (2px, 3px, or 4px)
4. Expand strokes before export (Object → Expand Appearance)
5. Export: File → Export As → SVG (for web/authoring) or PNG at 2× for raster

**Infographic for eLearning:**
1. Set artboard to 800×600 (standard eLearning canvas) or 1920×1080 (widescreen)
2. Create with Artboards — one per infographic panel
3. Use CC Libraries to share colors and styles with Photoshop and InDesign
4. Export all artboards: File → Export → Export for Screens → PNG/SVG

**Process diagram / flowchart:**
1. Use Illustrator's built-in shape libraries or create custom shapes
2. Connect with smart connectors via the Pen tool or Line Segment tool
3. Use Character/Paragraph Styles for text consistency
4. Export as SVG for interactivity hooks in authoring tools

## 14. Adobe InDesign for L&D

### Primary L&D Use Cases
- Participant workbooks and learner guides
- Facilitator / leader guides
- Job aids and quick reference cards
- Certificate of completion templates
- Program brochures and pre-work documents

### Core InDesign Concepts for L&D
- **Master Pages**: Set up headers, footers, page numbers, branding on Masters — applied to all pages automatically
- **Paragraph/Character Styles**: Define H1, H2, body, callout, caption styles — apply consistently; update once, changes everywhere
- **Object Styles**: For recurring frames (image frames, callout boxes, sidebars)
- **Text Variables**: Auto-updating fields (page number, date, section title in running header)
- **Linked Graphics**: Place images as links — update source file, InDesign updates automatically

### Accessible PDF Export from InDesign
1. Apply Paragraph Styles (not just visual formatting) — screen readers use style tags
2. Set reading order via Articles panel (Window → Articles) — drag to define screen reader sequence
3. Add alt text to all images (Object → Object Export Options → Alt Text)
4. Export: File → Export → Adobe PDF (Interactive or Print) → check "Create Tagged PDF" and "Use Structure for Tab Order"
5. Verify in Acrobat Pro: Accessibility → Full Check

## 15. Adobe After Effects for L&D

### L&D Use Cases
- Animated course intro/outro sequences
- Kinetic typography for key concept reveals
- Explainer animations (process flows, system diagrams)
- Lower thirds and title cards for video-based training
- Animated icons and micro-interactions (export as Lottie/JSON for eLearning)

### Core After Effects Concepts
- **Compositions**: The canvas — set to match output (1920×1080, 1280×720)
- **Keyframes**: Every animation is keyframes — position, scale, rotation, opacity over time
- **Easing**: Easy Ease (F9) on all keyframes — never linear animation in polished work; use Graph Editor for fine control
- **Pre-comps**: Nest compositions to organize complex animations; animate the pre-comp as a unit
- **Shape Layers vs. Footage**: Shape layers for vector animation; footage layers for video/image assets

### Lottie Export for eLearning (Animated Icons/Micro-interactions)
1. Design animation in After Effects using shape layers only (no raster)
2. Install Bodymovin plugin (free, via Adobe Extension Manager)
3. Export: Composition → Add to Render Queue or via Bodymovin → export as .json
4. Implement in eLearning: Use LottieFiles player in HTML widget within Storyline/Rise, or embed in web-based courses

## 16. Adobe Premiere Pro & Audition for L&D

### Premiere Pro — Video Editing Workflow for L&D

**Standard L&D video workflow:**
1. Ingest footage → organize in bins by module/scene
2. Rough cut on timeline — remove dead air, mistakes, long pauses
3. J-cuts and L-cuts for smooth transitions (audio leads or follows video)
4. Color correction: Lumetri Color → Basic Correction (exposure, contrast, white balance)
5. Add lower thirds and titles (use Motion Graphics templates for consistency)
6. Export: Media Encoder → H.264 → YouTube 1080p preset (good for LMS video upload)

**Screen recording editing in Premiere:**
- Remove filler words with Edit → Speech to Text → auto-generated transcript edit
- Speed ramp: Right-click clip → Speed/Duration for 1.25× playback (saves time, maintains comprehension)

### Audition — Audio Post-Production for L&D Narration

**Narration cleanup workflow:**
1. Open WAV/MP3 in Audition → Waveform Editor view
2. Noise reduction: Effects → Noise Reduction/Restoration → Capture Noise Print → Noise Reduction (apply 60-70%)
3. Remove room tone and silence: use Silence Selection on dead air sections
4. Normalize: Effects → Amplitude/Compression → Normalize to -3dB
5. DeEsser: Effects → Amplitude/Compression → DeEsser (reduce harsh sibilance)
6. Export: File → Export → File → WAV 44.1kHz/16-bit (for authoring tools) or MP3 192kbps (for web delivery)

## 17. Adobe XD for Learning UX/UI

### XD in the L&D Workflow
- Wireframe eLearning interfaces before building in authoring tools
- Create clickable prototypes for stakeholder/learner validation
- Design UI kits and component libraries for consistent course design
- Document design systems for L&D teams

### Core XD Concepts
- **Artboards**: One per screen/state — organize by flow (Welcome → Menu → Module 1 → Quiz)
- **Components (formerly Symbols)**: Reusable UI elements — buttons, nav bars, feedback panels; edit master, all instances update
- **Repeat Grid**: Rapidly create lists, card grids, icon rows — drag to repeat, override individually
- **Prototyping mode**: Wire artboards together with interactions (click, hover, drag, voice) — share link for stakeholder testing
- **Auto-Animate**: Morph between artboard states — demonstrates interaction behavior without building in Storyline

### Design System Documentation in XD
Create a "Design System" artboard set containing:
- Color swatches with hex values
- Typography scale (H1–caption with font, size, weight, line height)
- Button states (all four: normal, hover, active, disabled)
- Form elements (text input, dropdown, checkbox, radio)
- Icon library
- Spacing system (8px grid)
Share as a Cloud document with the dev/build team for handoff.

## 18. Adobe Acrobat Pro for L&D

### Key L&D Use Cases
- Create fillable PDF forms (pre-work assessments, reflection worksheets)
- Accessibility remediation for existing PDFs
- Combine course materials into organized PDF portfolios
- Protect sensitive documents with password encryption
- Add comments and review markup to storyboards and design documents

### Fillable Form Workflow
1. Prepare layout in InDesign or Word → export to PDF
2. Acrobat Pro: Tools → Prepare Form → Auto-detect form fields
3. Review and adjust auto-detected fields; add missing ones
4. Set field properties: tooltip (for accessibility), required/optional, validation
5. Tab order: Form tab → Fields → drag to set correct tab sequence
6. Distribute: Share as email attachment or via Acrobat Sign for tracked responses

### Accessibility Remediation
1. Tools → Accessibility → Full Check → review errors
2. Common fixes: Add alt text (Right-click image → Edit Alt Text), fix reading order (Tools → Reading Order panel), tag headings correctly, set document language (File → Properties → Advanced)
3. Re-run full check until zero errors in WCAG AA categories

## 19. Adobe Workflow Integration & CC Libraries

### CC Libraries — The L&D Designer's Superpower
- Store colors, character styles, graphics, and components in a shared library
- Accessible across ALL Adobe apps — set color in Illustrator, use it in Photoshop, InDesign, and XD
- Share libraries with team: team members get live updates when library assets change
- Practical use: Create "Course Brand Library" with all colors, fonts, icons → share with entire L&D team → consistency enforced automatically

### Linked Assets Workflow
- Place Illustrator graphics into InDesign/Premiere as **linked** files (not embedded)
- Update source → all placements update automatically
- Critical for: icon updates across a workbook, character revisions, logo changes

### Creative Cloud Express / Adobe Express
- Rapid asset creation for social learning, announcement graphics, job aid quick versions
- Templates pre-built — customize with brand colors/fonts from CC Libraries
- Good for: manager communication emails, cohort landing pages, learning campaign assets

### Export Presets for L&D Consistency
Set up and share export presets across the team:
- PNG for eLearning graphics: 144ppi, RGB, transparent background
- JPG for backgrounds: 85% quality, baseline standard, sRGB
- PDF for print workbooks: PDF/X-1a, CMYK, embedded fonts
- MP4 for LMS video: H.264, 1080p, AAC audio 192kbps
